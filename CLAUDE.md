## ⚡ ПЯТЬ ГЛАВНЫХ ПРАВИЛ (читать каждый раз)

1. **YOU MUST** при старте сессии: проверить `inbox/`, прочитать
   `docs/session-memory/` (последние 3-5 файлов), задать 4 вопроса онбординга.
   Подробности: `.claude/skills/onboarding.md`.

2. **YOU MUST** при архитектурном/дизайн-вопросе — искать в библиотеке через
   3-слойный поиск, не отвечать из общих знаний. Подробности:
   `.claude/skills/library-search.md`.

3. **YOU MUST** в конце сессии (или по команде "сохрани контекст") — записать
   `docs/session-memory/<date>-<role>-<topic>.md`. Подробности:
   `.claude/skills/session-memory.md`.

4. **YOU MUST** при команде "разбери inbox" — спросить "один проект или
   несколько?" и группировать по проекту, а не по типу файла. Подробности:
   `.claude/skills/inbox-triage.md`.

5. **YOU MUST** при сложном решении применить Принцип #0 Алексея (Простота):
   спросить "как обычные люди решают это самым простым и дешёвым способом?".
   Если ответ — "сложная архитектура / 5 сервисов" — откатиться и упростить.
   См. `kanon/alexey-11-principles.md`.

## 📋 ПРАВИЛА ПОВЕДЕНИЯ (всегда)

- **Никакого маркетинг-стиля** — не pitch, не продавать решения, только факты + альтернативы
- **Краткость в чате** — ответ длиннее ~3 КБ → спроси разрешения разделить.
  Не показывай полные файлы кода в чате — делай Edit/Write, в чат summary.
- **Не лезь в репо самостоятельно** — без команды читай только: `kanon/`,
  `CLAUDE.md`, `.claude/skills/*.md`, `alexey-materials/metadata/index_compact.json`.
  Не делай `find`/`ls`/`grep` по подпапкам без разрешения.
- **Жёсткий коридор разрешений** уже задан в `~/.claude/settings.json`:
  `rm*`, `sudo`, `git push --force`, `git reset --hard` — **запрещены**.
  `git push`, `npm publish`, `docker build` — **спрашивают**.
