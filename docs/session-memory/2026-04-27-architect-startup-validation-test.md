# Главная находка — устаревший снимок CLAUDE.md

Тестовый чат **процитировал старый текст**, которого нет в репо после `d869ace`:

| Цитата чата | Что сейчас в файле |
|---|---|
| «Лимит ~3 КБ» | 10 000 знаков |
| «5 YOU MUST» | 4 YOU MUST |
| «Имя нигде не указано» | Первой строкой в «Правилах поведения» |
| «handoff НЕ ЗНАЮ» | Карта папок + правило #3 + межчат-блок |
| «Слой 0 НЕ ЗНАЮ» | Правило #2 |

**Корень:** SessionStart hook не делает `git pull`. Чаты, открытые ДО
коммита `d869ace`, работают на снимке репо на момент **их старта**. Свежие
правила не достигают.

# Решения и причины

1. **Не добавлять `git pull` в hook** — сознательное решение (по дабл-чеку):
   - `git pull` во время сессии может вызвать merge-конфликт
   - Не-программист застрянет на конфликте
   - Hook должен быть быстрым и безопасным

2. **Перезапуск сессии — единственный способ обновить снимок.** Зафиксировано
   в `troubleshoot/stale-snapshot.md` как пользовательская практика.

3. **Подсматривал ли тестовый чат — НЕТ.** Признаки честности:
   - 5 явных «НЕ ЗНАЮ»
   - В п.8 признал «я выудил #5 Fail Loud из формулировки вопроса»
   - Расхождение «11 vs 12» отметил без подстройки
   - Если бы подсматривал — знал бы Слой 0 и принципы по именам
