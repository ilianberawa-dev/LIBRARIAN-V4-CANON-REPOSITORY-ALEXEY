**Архитектуру с нуля строить не надо.** Используем стандартный проверенный стек: **Claude Code + OpenClaw + LightRAG + Supabase**. Это то же, что я рекомендую всем клиентам — надёжно, понятно, масштабируется.

## Главный принцип

**Тебе не нужны мульти-агентные системы, Paperclip и «AI-компании».** Нужен простой флоу:

```
Claude Code (dev на ноуте) → пишешь и тестишь СКИЛЛЫ →
закидываешь их в OpenClaw → OpenClaw запускает их автономно по крону.
```

Всё. Один агент + набор скиллов + крон. Никаких оркестраторов, критиков, пирамид агентов — это только жрёт токены и ломается.

### Почему ты мучался неделю с OpenClaw

- Локалка + openclaw.gdn — это плохая связка для прода: домашний интернет, электричество, руками возиться.
- Watchdog жрёт токены потому что плохо настроен.
- Правильное решение: **VPS за копейки, Docker, Claude Code для разработки скиллов, OpenClaw только для автономного прогона готовых скиллов.**

## Агенты vs скиллы — как это устроено

Это место где все путаются. Разведём.

### Три слоя: агент → скилл → инструмент

```
АГЕНТ   = модель + системный промпт + тулы
          Claude Code знает "как работать в роли"
           ↓ использует ↓
СКИЛЛ   = markdown-инструкция под конкретную задачу
          "как сделать вот эту штуку"
           ↓ использует ↓
ИНСТРУМЕНТ = MCP / bash / HTTP API
             физическое действие
```

### Глобальные субагенты (настроил раз — работают везде)

В `~/.claude/agents/` лежат три универсала, разделённые по модели:

| Агент | Модель | Зачем |
|---|---|---|
| **scout-haiku** | Haiku | Быстрый поиск: файлы, LightRAG-запросы, справки, подсчёты. Дёшево. |
| **worker-sonnet** | Sonnet | Основной исполнитель: пишет скиллы, парсеры, обработку. Дефолт. |
| **strategist-opus** | Opus | Глубокое мышление: архитектура, планы, трейдоффы. Редко. |

Когда пишешь в Claude Code «сделай мне парсер для Rumah123» — главная сессия сама роутит задачу нужному субагенту. Тебе не надо думать какую модель юзать — CLAUDE.md решает.

**Экономия:** 80% простых задач уходят на Haiku (в 15 раз дешевле Opus).

### Скиллы — доменные, под твою задачу

Скилл = markdown с чёткой инструкцией «что делать». Пишешь один раз, потом и Claude Code и OpenClaw используют один и тот же файл.

Под твой кейс (аналитика недвижимости Бали):

| Скилл | Что делает |
|---|---|
| **parse_listings_web** | Парсит открытые площадки (Rumah123, OLX, Balirealty). Playwright или httpx. Пишет в `raw_listings`. |
| **normalize_listing** | Берёт сырое объявление → Claude нормализует: тип, район, цена в USD/год, спальни, контакт. Пишет в `properties`. |
| **store_to_supabase** | Дедупликация по URL+адрес+цена. Обновляет если уже есть. |
| **search_properties** | Поиск по фильтрам (район, тип, цена, спальни). Возвращает список. |
| **market_snapshot** | Раз в неделю считает медианы по районам, тренды. Пишет в `market_snapshots` + в LightRAG как текст. |

**Важно:** Telegram-источники НЕ делаем вообще на этом этапе. Ни парсинг каналов, ни бот для общения. Сначала отладь веб-парсинг и накопление чистых данных — интерфейс поверх этого (дашборд, бот, что угодно) собирается потом за пару часов.

### Модель для OpenClaw

OpenClaw модель-агностик. На продакшне тяжёлые модели **не нужны** — скилл задаёт жёсткий детерминизм, модели остаётся только следовать инструкции и не выдумывать лишнего.

Варианты (от дешевле к дороже):

- **Qwen 3.5 (самая маленькая)** — рабочий пример: логистическая компания с несколькими автопарками использует именно её. Прекрасно работает на детерминированных скиллах. Дешевле MiniMax.
- **MiniMax M2.7** — рекомендация по умолчанию. Алексей лично использует именно эту модель на своих агентах. ~в 15 раз дешевле Opus. Хорошее соотношение цена/качество.
- **OpenAI (gpt-5.1-mini)** — аналогичный ценник, быстрая.
- **Claude API (Sonnet 4.6)** — через твой же Claude-ключ. Лучшее качество, но дороже. ~$15-30/мес на старте. Имеет смысл если скиллы сложные с большой вариативностью входных данных.

**Ключевой принцип:** для разработки и тестирования скиллов (в Claude Code) нужны сильные модели (Opus/Sonnet). Для прода — достаточно дешёвых. Чем жёстче скилл + скрипты → тем меньше шансов накосячить → тем дешевле можно обойтись.

## Скиллы — детально (что писать первым)

### 1. parse_listings_web (приоритет 1)

**Вход:** имя источника (rumah123, olx, balirealty, …).
**Что делает:**
- Заходит на сайт, проходит по страницам листинга.
- Для каждого объявления достаёт raw HTML + текст + картинки (ссылки).
- Пишет сырыми в `raw_listings` со статусом `pending`.
- Обновляет `sources.last_parsed_at`, `total_parsed`.

**Инструмент:** Playwright (для JS-сайтов) или httpx + BeautifulSoup (для статичных).

### 2. normalize_listing (приоритет 1)

**Вход:** `raw_listings.id` со статусом `pending`.
**Что делает:**
- Берёт сырой текст объявления.
- Через Claude извлекает структурированные поля по JSON-схеме:
  ```json
  {
    "type": "villa | apartment | land | house",
    "district": "Canggu | Seminyak | ...",
    "bedrooms": 3,
    "bathrooms": 2,
    "area_m2": 120,
    "price_usd_yearly": 15000,
    "rental_period": "yearly | monthly | freehold | leasehold",
    "lease_years": 25,
    "contact": {...}
  }
  ```
- Если извлечение не удалось — статус `failed` + error message.
- Если ок — вызывает `store_to_supabase`.

**Важно:** JSON-схема как structured output — модель не «болтает», возвращает строго формат.

### 3. store_to_supabase (приоритет 1)

**Вход:** нормализованный объект.
**Что делает:**
- Дедуп по `source_url + district + price_usd_yearly`.
- Если новый — INSERT + флаг «новый объект» для алертов.
- Если есть — UPDATE цены/даты, если изменилось — отметить `price_changed_at`.

### 4. market_snapshot (приоритет 2)

**Вход:** нет (крон по воскресеньям).
**Что делает:**
- По каждому району считает: медиана, мин, макс, кол-во объявлений, тренд vs прошлая неделя.
- Пишет в `market_snapshots`.
- Составляет текстовое описание → сохраняет в LightRAG (`insert_text`).
- Со временем LightRAG накапливает «историю рынка» — полезно для стратегии.

### 5. search_properties (приоритет 2)

**Вход:** фильтры от пользователя.
**Что делает:**
- SELECT из Supabase по фильтрам.
- Возвращает список JSON-ом или текстом.
- Используется из команд в Claude Code. Интерфейс для клиентов (дашборд, бот) — отдельная задача позже.

## Таблицы Supabase (можно отдать скилл-креатору на автосоздание)

```sql
CREATE TABLE raw_listings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  source_name TEXT NOT NULL,
  source_url TEXT,
  raw_text TEXT NOT NULL,
  raw_html TEXT,
  status TEXT DEFAULT 'pending',  -- pending/processed/failed/duplicate
  property_id UUID,
  error_message TEXT,
  parsed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE properties (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  type TEXT NOT NULL,                -- villa/apartment/land/house
  district TEXT,                     -- Canggu/Seminyak/Ubud/...
  address TEXT,
  bedrooms INTEGER,
  bathrooms INTEGER,
  area_m2 NUMERIC,
  land_area_m2 NUMERIC,
  price_usd NUMERIC,
  price_original TEXT,
  rental_period TEXT,                -- yearly/monthly/freehold/leasehold
  lease_years INTEGER,
  contact_name TEXT,
  contact_phone TEXT,
  contact_whatsapp TEXT,
  source_url TEXT,
  source_name TEXT,
  image_urls TEXT[],
  raw_text TEXT,
  is_active BOOLEAN DEFAULT true,
  first_seen_at TIMESTAMPTZ DEFAULT now(),
  last_seen_at TIMESTAMPTZ DEFAULT now(),
  price_changed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_properties_district ON properties(district);
CREATE INDEX idx_properties_type ON properties(type);
CREATE INDEX idx_properties_price ON properties(price_usd);
CREATE INDEX idx_properties_active ON properties(is_active);

CREATE TABLE market_snapshots (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  snapshot_date DATE NOT NULL DEFAULT CURRENT_DATE,
  district TEXT,
  total_listings INTEGER,
  new_listings_week INTEGER,
  median_price_usd NUMERIC,
  avg_price_usd NUMERIC,
  min_price_usd NUMERIC,
  max_price_usd NUMERIC,
  price_trend TEXT,
  price_change_pct NUMERIC,
  breakdown JSONB,
  summary_text TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE sources (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL UNIQUE,
  type TEXT NOT NULL,                -- website/rss
  url TEXT,
  config JSONB DEFAULT '{}',
  parse_interval_minutes INTEGER DEFAULT 180,
  is_active BOOLEAN DEFAULT true,
  last_parsed_at TIMESTAMPTZ,
  total_parsed INTEGER DEFAULT 0,
  error_count INTEGER DEFAULT 0,
  last_error TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

```

## Что НЕ делаем

- ❌ **Paperclip** и «AI-компании». Не нужно. Ты застрял именно потому что пытался их поднять.
- ❌ **Мульти-агентные системы** (оркестратор + аналитик + стратег + критик). Жрут токены, ломаются, обслуживание дороже пользы.
- ❌ **OpenClaw на локалке через openclaw.gdn.** Не работает стабильно — это не тебе не везёт, это локалка + криво завёрнутая сборка. Переезд на VPS это закрывает.
- ❌ **Facebook Marketplace.** Банит, требует авторизацию, меняет структуру. Забудь на старте.
- ❌ **Telegram вообще** — ни парсинг каналов, ни бот для общения на этом этапе. Фокус на веб-источники и чистые данные. Интерфейс делается в последнюю очередь когда данные уже есть.
- ❌ **Писать скилл сразу в OpenClaw.** Сначала в Claude Code, прогнал руками, увидел результат — только потом в OpenClaw.
- ❌ **«Саморазвивающаяся система, которая сама всё пишет».** Это маркетинговая магия. Реальность: скиллы пишешь ты (с помощью Claude Code), система накапливает ДАННЫЕ и ЗНАНИЯ, становится умнее за счёт этого.

## Дорожная карта

### Этап 1 — Фундамент (день консультации + 1 день после)

1. Купить VPS в азиатской локации — **Linode Jakarta** или **Vultr Singapore** (~$5-6/мес). Запасной вариант — [AdminVPS «Старт»](https://my.adminvps.ru/aff.php?aff=28573&url=vps/vps_russia.php) (~1300 ₽/мес, промокод `AK-PRIVATE60`), но локации у него только европейские.
2. Поставить Docker + Docker Compose.
3. Docker Compose с тремя сервисами: Supabase / LightRAG / OpenClaw.
4. Подключить Supabase MCP и LightRAG MCP к Claude Code.
5. Проверить что Claude Code ходит в обе.

> По Supabase MCP: у меня для ребят из [привата](https://t.me/tribute/app?startapp=sr3L) уже развёрнут общий сервер — можно не поднимать свой с нуля, Claude Code ходит туда напрямую и агент видит базу. Там же кидаю инструкции по подключению и заметки по ходу — когда что-то новое нащупываю в Claude Code / OpenClaw / скиллах, оформляю и выкладываю.
> По самому Claude Code: попробуй **десктоп-приложение** (не только терминал). Я сам недавно вернулся к нему — как ни странно, по ощущениям удобнее: окно, история, нормальная вставка картинок и файлов. Терминал тоже ок, просто для сравнения поработай денёк в десктопе — возможно зайдёт больше.

## SaaS-потенциал (на будущее)

Supabase — правильный выбор для SaaS:
- Авторизация встроена
- Прикручиваешь платёжку поверх
- Дашборд/фронт цепляется как обвязка к уже готовой базе данных
- Масштабируется без переезда

**Важно:** интерфейс (дашборд, бот) делается В ПОСЛЕДНЮЮ ОЧЕРЕДЬ — когда данные уже накоплены и понятно ЧТО показывать пользователям. Сначала данные, потом интерфейс.

Портфель Ильи (потенциальные рынки для расширения): Дубай, Сочи, Бали. Архитектура на Supabase позволяет добавить новые рынки без переработки — просто новые источники в таблице `sources` и новые значения в поле `district`/локация.
