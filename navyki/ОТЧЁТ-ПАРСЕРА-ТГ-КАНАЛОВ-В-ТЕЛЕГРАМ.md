# Отчёт парсера ТГ-каналов в Telegram

**Назначение:** Регулярные отчёты о прогрессе скачивания и транскрипции контента из ТГ-канала. Шлются автоматически в Telegram через `cron` каждые 2-6 часов.

**Принцип:** RUNTIME-данные из источников истины — НИКАКОГО хардкода цифр.

**Связанные файлы:**
- `navyki/heartbeat-telegram-pattern.md` — Self-healing watchdog (Heartbeat pattern)
- `navyki/claude-bot-parser-control.md` — Управление парсером через Claude Desktop
- `troubleshoot/tg-parser-state.md` — Текущее состояние и известные проблемы
- `scripts/tg_notify_v2.sh` — Эталонный код отчёта

## Философия отчёта

**Одна цифра — ложь. Категории — правда.**

❌ Плохо: `Скачано: 48/117`  
✅ Хорошо:
```
P1 ▓▓▓▓▓▓▓▓▓▓ 100%  (скрипты/конфиги: 27/27)
P2 ▓▓▓▓▓▓░░░░  62%  (PDF/доки: 4/7)
P3 ▓▓▓▓▓▓▓▓░░  81%  (фото: 13/16)
P4 ░░░░░░░░░░   0%  (видео: 0/67)
```

**Почему это важно:**
- P1 100% при P4 0% = разные приоритеты, разные действия
- Missing list конкретен: какие `msg_id` не скачаны, сколько GB осталось
- Источник истины = `library_index.json`, не лог-файл (лог может потеряться)

**Три кита отчёта:**
1. **Ожидаемое** — что ДОЛЖНО быть (из `library_index.json`)
2. **Сделанное** — что РЕАЛЬНО есть (из файловой системы / БД)
3. **Missing** — разница (список `msg_id`, размер очереди в GB)

## Источники данных (5 источников истины)

| Источник | Что даёт | Как читать |
|---|---|---|
| `library_index.json` | Список ВСЕХ постов канала (ожидаемое) | `jq '[.[] \| select(.media != null)] \| length'` |
| `transcripts/` в репо | Что реально транскрибировано | `ls alexey-materials/transcripts/*.json \| wc -l` |
| `/opt/tg-export/transcripts/` | Локальные транскрипты на сервере | `ls \| wc -l` |
| `alexey-materials/media/` | Документы (PDF/фото) в репо | `ls \| wc -l` |
| Postgres `telegram_transcripts` | Итоговое хранилище (Supabase) | `SELECT COUNT(*) FROM telegram_transcripts` |

**Правило:** `library_index.json` — единственный источник истины для «ожидаемого».  
Лог-файлы (`download.log`) — вторичный источник, используется только для прогресс-баров внутри сессии.

## Категоризация (тип → приоритет → источник проверки)

| Тип контента | Расширение | Приоритет | Проверка наличия |
|---|---|---|---|
| Скрипты / конфиги | `.py`, `.sh`, `.json`, `.yaml` | P1 | `ls media/ \| grep -E '\.(py\|sh\|json\|yaml)$'` |
| Документы / PDF | `.pdf`, `.docx` | P2 | `ls media/ \| grep -E '\.(pdf\|docx)$'` |
| Фотографии | `.jpg`, `.png` | P3 | `ls media/ \| grep -E '\.(jpg\|png)$'` |
| Видео / аудио | `.mp4`, `.wav`, `.m4a` | P4 | `ls transcripts/ \| grep '.transcript.json'` |

**Видео не хранить в репо** — только транскрипты (`.transcript.json` + `.transcript.txt`).

## Прогресс-бар: функция `render_bar`

```bash
render_bar() {
  local pct=$1
  local blocks=$((pct / 10))
  local bar=""
  local i
  for ((i=0; i<blocks; i++)); do bar+="▓"; done
  for ((i=blocks; i<10; i++)); do bar+="░"; done
  echo "$bar"
}

# Использование:
PCT=75
BAR=$(render_bar $PCT)
echo "P1 ${BAR} ${PCT}%"
# → P1 ▓▓▓▓▓▓▓░░░ 75%
```

**Правило:** 10 блоков = 100%, каждый блок = 10%. Округление вниз (`/10`).

## Полный код: `scripts/tg_notify_v2.sh` (эталон из продакшна)

```bash
#!/bin/bash
# Telegram notification - IMPROVED VERSION v2
# Цветная визуализация прогресса с процентами
# RUNTIME данные из источников истины — без хардкода

set -u

BASE=/opt/tg-export
ENV="$BASE/.env"

if [ ! -f "$ENV" ]; then exit 0; fi
set -a; source "$ENV"; set +a

BOT_TOKEN="${BOT_TOKEN:-}"
CHAT_ID="${CHAT_ID:-}"

if [ -z "$BOT_TOKEN" ] || [ -z "$CHAT_ID" ]; then
  exit 0
fi

# Status snapshot
S_FILE="$BASE/_status.json"
[ ! -f "$S_FILE" ] && bash "$BASE/heartbeat.sh"
S=$(cat "$S_FILE")

# Times
BALI_TIME=$(TZ='Asia/Makassar' date '+%H:%M %d.%m')
UTC_TIME=$(date -u '+%H:%M')

# Grok STT stats — RUNTIME из реальных файлов
TOTAL_SEC=0
for f in "$BASE/transcripts/"*.transcript.json; do
  [ -f "$f" ] || continue
  case "$f" in *_part*) continue ;; esac
  d=$(jq -r '.duration // 0' "$f" 2>/dev/null)
  TOTAL_SEC=$(echo "$TOTAL_SEC + $d" | bc)
done
GROK_COST=$(echo "scale=2; $TOTAL_SEC / 3600 * 0.10" | bc 2>/dev/null || echo "0")
TOTAL_HOURS=$(echo "scale=1; $TOTAL_SEC / 3600" | bc 2>/dev/null || echo "0")

# Download progress — из лога текущей сессии
P1_OK=$(grep -c '\[ok P1\]\|\[skip P1\]' "$BASE/download.log" 2>/dev/null || echo 0)
P2_OK=$(grep -c '\[ok P2\]\|\[skip P2\]' "$BASE/download.log" 2>/dev/null || echo 0)
P3_OK=$(grep -c '\[ok P3\]\|\[skip P3\]' "$BASE/download.log" 2>/dev/null || echo 0)

# RUNTIME totals из library_index.json (не хардкод!)
P1_TOTAL=$(jq '[.[] | select(.priority == "P1")] | length' "$BASE/library_index.json" 2>/dev/null || echo 27)
P2_TOTAL=$(jq '[.[] | select(.priority == "P2")] | length' "$BASE/library_index.json" 2>/dev/null || echo 7)
P3_TOTAL=$(jq '[.[] | select(.priority == "P3")] | length' "$BASE/library_index.json" 2>/dev/null || echo 16)

# Защита от деления на ноль
[ "$P1_TOTAL" -eq 0 ] && P1_TOTAL=1
[ "$P2_TOTAL" -eq 0 ] && P2_TOTAL=1
[ "$P3_TOTAL" -eq 0 ] && P3_TOTAL=1

P1_PCT=$((P1_OK * 100 / P1_TOTAL))
P2_PCT=$((P2_OK * 100 / P2_TOTAL))
P3_PCT=$((P3_OK * 100 / P3_TOTAL))

render_bar() {
  local pct=$1
  local blocks=$((pct / 10))
  local bar=""
  local i
  for ((i=0; i<blocks; i++)); do bar+="▓"; done
  for ((i=blocks; i<10; i++)); do bar+="░"; done
  echo "$bar"
}

P1_BAR=$(render_bar $P1_PCT)
P2_BAR=$(render_bar $P2_PCT)
P3_BAR=$(render_bar $P3_PCT)

DL_STATUS=$(echo "$S" | jq -r '.download.status')
DL_IDLE=$(echo "$S" | jq -r '.download.idle_sec')
TX_STATUS=$(echo "$S" | jq -r '.transcribe.status')
TX_COUNT=$(echo "$S" | jq -r '.transcripts_count')
MEDIA_FILES=$(echo "$S" | jq -r '.media_files')
MEDIA_MB=$(echo "$S" | jq -r '.media_bytes' | awk '{printf "%.0f", $1/1048576}')

MSG="🌴 <b>TG-Export ${BALI_TIME}</b> (UTC ${UTC_TIME})

📥 <b>Download:</b> ${DL_STATUS} (idle ${DL_IDLE}s)

  <code>P1 ${P1_BAR} ${P1_PCT}%</code>
  scripts/configs: ${P1_OK}/${P1_TOTAL}

  <code>P2 ${P2_BAR} ${P2_PCT}%</code>
  PDFs/docs: ${P2_OK}/${P2_TOTAL}

  <code>P3 ${P3_BAR} ${P3_PCT}%</code>
  photos: ${P3_OK}/${P3_TOTAL}

🎙 <b>Transcribe (Grok STT):</b> ${TX_STATUS}
  Files done: ${TX_COUNT}
  Audio: ${TOTAL_HOURS}h (${TOTAL_SEC}s)
  💰 Cost: \$${GROK_COST}

💾 Media: ${MEDIA_FILES} files / ${MEDIA_MB} MB
"

RESPONSE=$(curl -sS -X POST "https://api.telegram.org/bot${BOT_TOKEN}/sendMessage" \
  --data-urlencode "chat_id=${CHAT_ID}" \
  --data-urlencode "text=${MSG}" \
  -d "parse_mode=HTML" 2>&1)

if echo "$RESPONSE" | jq -e '.ok' >/dev/null 2>&1; then
  echo "[$(date -u +%FT%TZ)] notify_v2: sent OK" >> "$BASE/heartbeat.log"
else
  echo "[$(date -u +%FT%TZ)] notify_v2: FAIL $RESPONSE" >> "$BASE/heartbeat.log"
fi
```

## Адаптация под другой канал (4 шага)

1. **Анализ:** Какие типы медиа в канале? Сколько P1/P2/P3/P4? → обновить категоризацию
2. **Категории:** Переименовать метки в `MSG` (P1/P2/P3 → нужные имена)
3. **Код:** Изменить фильтры `jq '.[] | select(.priority == "P1")'` под свою структуру `library_index.json`
4. **Заголовки:** Изменить `🌴 TG-Export` → `🎯 Имя-канала-Export`

## 6 известных граблей (решены)

| # | Проблема | Симптом | Решение |
|---|---|---|---|
| 1 | `cut -c` режет байты, не символы | UTF-8 кириллица → Telegram отвергает сообщение | Использовать `python3 -c "print(s[:N])"` для усечения |
| 2 | `comm` требует лексикографическую сортировку | `comm: file 2 is not in sorted order` | `sort -V` для версионных строк, `sort` для обычных |
| 3 | SSH + double-quotes ломает экранирование | Переменные не раскрываются в удалённой команде | Heredoc или single-quote с eval на удалённой стороне |
| 4 | Хардкод `P1_TOTAL=27` протухает | После добавления постов % > 100% | Читать из `library_index.json` через `jq` (RUNTIME) |
| 5 | Fail без лога | `curl` молчит при ошибке API | `echo "FAIL $RESPONSE" >> heartbeat.log` обязательно |
| 6 | Деление на 0 | `P1_TOTAL=0` → bash ошибка | `[ "$P1_TOTAL" -eq 0 ] && P1_TOTAL=1` защита |
