# Симптом

Тестовый чат отвечает на вопросы про CLAUDE.md / skills цитатами,
которых УЖЕ НЕТ в файле. Например:

- Говорит «лимит ~3 КБ» — а в файле сейчас «10 000 знаков»
- Говорит «5 правил YOU MUST» — а в файле 4
- Говорит «имя нигде не указано» — а в файле первой строкой «Имя — Илья»

# Причина

SessionStart hook **не делает** `git pull`. Это сознательное решение:
`git pull` во время сессии может дать merge-конфликт, не-программист
застрянет на `<<<<<<< HEAD`.

Поэтому каждый чат работает на снимке репо **на момент его старта**.
Если CLAUDE.md / skills обновлены в другом чате и запушены — старые
сессии **не видят** свежих правил.

# Решение

| Симптом | Действие |
|---|---|
| Чат цитирует старые формулировки правил | `/clear` (перезапуск сессии) или открыть новый чат |
| Hook на старте печатает мало session-memory | `git pull` в Bash + `/clear` |
| Файл памяти видим в `ls`, но чат говорит «не вижу» | Перезапуск сессии |

**Самый надёжный способ** — закрыть чат и открыть новый. SessionStart
hook отработает на свежем снимке.

# Когда это норма (не нужно перезапускать)

- Чат сделал коммит и работает с тем что сам положил — он видит свои
  изменения сразу
- Изменения вне `.claude/` и `CLAUDE.md` — не влияют на стартовый ритуал

# Анти-паттерн

- ❌ Добавлять `git pull` в SessionStart hook — риск merge-конфликта
- ❌ Объяснять Илье что нужно `git fetch && git reset --hard origin/main` —
  это запрещено коридором разрешений (и опасно)
- ✅ Просто перезапустить сессию
