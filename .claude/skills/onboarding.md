**Применяется когда:** в начале **каждой** сессии Claude Code (включая `/resume`,
`/compact`, `/clear`) — до любого действия. Исключение: задача из первого
сообщения очевидно узкая (одна команда / один файл) — выполняй молча.

# Онбординг — ритуал первого сообщения

## ШАГ 0 — ПРОВЕРКА INBOX (до всех вопросов)

```bash
ls inbox/ 2>/dev/null | grep -v '^.gitkeep$\|^README.md$'
```

- **Если есть файлы** — упомяни в первом сообщении:
  > «📥 В inbox/ лежит N файлов: [перечисли]. Разобрать сейчас или после задачи?»
- **Если пусто** — молчи про inbox

## ШАГ 1 — 4 вопроса в одном сообщении

1. **Какая моя роль сегодня?**
   - А) Архитектор / большая задача
   - Б) Узкий установщик (одна команда — например "сделай алиас")
   - В) Не-программирование
   - Г) Дизайн / UI
2. **Создать ли алиас `sync`** для этого репо? (`git pull && ls inbox/`)
3. **Есть что-то загрузить в `inbox/`** (единая свалка, разберём вместе)?
4. **Подтверждение коридора:** "Я понимаю что не могу делать `rm`, `sudo`,
   `git push --force`, `git reset --hard`, и пишу только в этом репо. На push,
   создание PR и опасное — спрашиваю. Идём?"

После ответа — **подтверди роль вслух одной строкой** и работай. Не молчи.

**ИСКЛЮЧЕНИЕ:** если задача из первого сообщения **очевидно узкая** (одна
команда, один файл) — просто выполняй, не задавай 4 вопроса.

## Перед стартом — прочитать предыдущие сессии

SessionStart hook печатает **3 последних** файла из `docs/session-memory/`.
Прочитай **ровно эти 3** — без фильтрации по роли. Свежий файл может закрывать
техдолги старого; пропустишь — процитируешь устаревшие задачи как открытые.

**Техдолги из session-memory могут быть уже закрыты** — сверь с `CLAUDE.md` /
`.claude/settings.json` прежде чем называть их открытыми.

Если нужен файл старше трёх напечатанных hook — читай через `Read` по имени напрямую.
Эти 3 — точка входа, не ограничение.

В подтверждении упомянуть: *"Прочитал предыдущие сессии: X, Y, Z. Продолжаю с..."*

## Роли — детали поведения

### 🟦 Архитектор / большая задача
- Читает `_INDEX.md` (полное дерево библиотеки + инвентарь skills)
- Читает `kanon/alexey-11-principles.md` (фон) + `kanon/memory-pyramid-principle.md`
- Применяет 3-слойный поиск (см. skill `library-search`)
- Решения опираются на принципы Алексея + материалы из библиотеки
- Всегда ссылается на источники

### 🟨 Узкий установщик (одна команда)
- 11 принципов — фон, не директива
- Делает задание буквально и узко
- НЕ читает библиотеку (только если задание явно требует)
- Останавливается **только** при нарушении канона на русском:
  *"Это нарушает Принцип #3 (Simple Nodes) — продолжаем?"*

### 🟩 Не-программирование
- Сразу спрашивает: "О чём задача? Каким разделом библиотеки пользоваться?"
- Без ответа не действует
- 11 принципов не насаждает

### 🟪 Дизайн / UI
- Читает `_INDEX.md` (полное дерево библиотеки + инвентарь skills)
- Поведение архитектора + читает раздел `dizain/`
- Особое внимание Принципу #0 (Простота) и пирамиде памяти
- Проверяет: загружено ли что-то в `inbox/` для разбора

## Web-Claude handoff (передача файлов через JSON)

**Триггер:** в первом сообщении сессии есть JSON-блок с полями
`from: "web-claude"`, `branch`, `files[]`, `verify_command`.

Это передача артефактов из claude.ai web (где нет git/MCP/реального sha256) в
этот репо. Web-Claude **не умеет** считать криптографические хеши — в
манифесте надёжны только `bytes`, `lines`, `first_100`, `last_100`. **Sha256
не сверяй** — оно галлюцинируется.

**Протокол приёма:**
1. `git fetch && git diff main..origin/<branch> --name-only` — список файлов
   совпадает с `files[]` манифеста?
2. Для каждого файла: `wc -c`, `wc -l`, `head -1`, `tail -1` — сверь с
   `bytes/lines/first_100/last_100`. **Вывод bash в чате обязателен.**
3. Выполни `verify_command` (обычно grep по заглушкам), вывод stdout в чат.
4. Финальный отчёт: фактические цифры из bash + явный дисклеймер:
   **«Статика чиста, реальный прогон/sandbox НЕ делался — интеграционный тест
   отдельно».**
5. **НЕ мержи в main без явного "го".** Принцип #5 Алексея — Fail Loud.

Подробности про антигаллюцинации — `.claude/skills/state-check.md`.
