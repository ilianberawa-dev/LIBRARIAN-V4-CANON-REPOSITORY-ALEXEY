**Применяется когда:** пользователь задаёт операционный или уточняющий вопрос
("куда?", "что с X?", "проверь", "почему N не работает?", "сделай Y"); ИЛИ
получаешь handoff с описанием состояния от другого чата. **До clarifying
вопроса — обязательны проверки.**

# Принцип

**Не галлюцинируй контекст. Проверь состояние через инструменты.**

LLM по умолчанию генерирует, не наблюдает. Перед уточнением «что вы имеете в
виду?» сначала прочитай очевидные источники истины. У тебя есть Bash, Read,
MCP — используй их.

# 3 обязательные проверки до уточнения

При любом операционном/уточняющем вопросе сделай **в таком порядке**:

1. **`git fetch && git log HEAD..@{u} --oneline`** — что прилетело с remote с
   прошлой сессии (не локаль — remote!)
2. **`git log --oneline -20`** — недавняя активность локально
3. **При наличии MCP**: `mcp__github__list_issues --label handoff --state open`
   — открытые передачи между чатами

Если ответ найден в этих трёх — **НЕ задавай уточняющий вопрос**. Отвечай по
факту, ссылаясь на конкретный коммит / issue.

Если ответ не найден — задавай вопрос, **упомянув** что уже проверено:
> «Я посмотрел git log за 20 коммитов и issues с label handoff, ответа там
> нет. Уточни: …»

# Запрет на симуляцию OK

**ЗАПРЕЩЕНО** писать «проверил», «OK», «соответствует», «всё корректно»,
«файл полный», «заглушек нет» — пока в **этом же** сообщении не виден вывод
Bash/Read tool, подтверждающий факт.

Это лечит три типа галлюцинаций:
- (а) прочитал только заголовки → решил что всё ок
- (б) вообще не прочитал → отчитался по содержимому
- (в) прочитал часть → достроил остальное «из общих знаний»

# При проверке полученных файлов

Если получаешь файлы от другого чата (web-Claude handoff, преемник,
пользователь):

- **`wc -c`, `wc -l`, `head -1`, `tail -1`** — сверь с манифестом, если есть
- **`grep -nE "TODO|FIXME|XXX|<TBD>|lorem|заглушка|placeholder"`** — ловит
  явные заглушки
- **НЕ ловит «правдоподобный фейк»** — код, который выглядит реалистично, но
  зовёт несуществующий API
- В отчёте обязателен дисклеймер:
  > «Статика чиста, реальный прогон/sandbox НЕ делался — интеграционный тест
  > нужен отдельно.»
- **Sha256 от web-Claude не сверяй** — он галлюцинируется (нет crypto-libs в
  Analysis Tool у claude.ai web)

# Принцип #5 Алексея — Fail Loud

При **любом** сомнении (расхождение цифр, отсутствует файл, маркер не виден,
`curl`/`git fetch` упал) — **STOP**. Не продолжай, не достраивай, не
«угадывай». Доложи что именно сломалось и где.

# Канон vs чат

**Канон** = только `kanon/` (11 принципов + memory-pyramid + simplicity-first)
+ `alexey-materials/` (артефакты автора Алексея Колесова). Каждый канонический
файл имеет фронт-матер с `author:` / `source_msg_id:` / `extracted_from:`.

**Слова Ильи в чате** — это запросы и решения **пользователя**, не канон.

**Запрещено:**
- Цитировать фразы Ильи как «принцип Алексея» / «канон Алексея»
- Писать в коммитах/памяти «Алексей сказал» имея в виду фразу из чата
- Приписывать Илье статус канонического автора

**При записи в session-memory** различать:
- «решено в чате с Ильёй» — для решений пользователя
- «согласно `kanon/<file>:<line>`» — для ссылок на артефакт автора
