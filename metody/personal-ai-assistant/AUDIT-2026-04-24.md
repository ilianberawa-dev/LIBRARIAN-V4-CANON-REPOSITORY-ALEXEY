**Дата:** 2026-04-24
**Метод:** self-audit по канону + industry patterns
**Результат:** 8 находок → упрощённый MVP v1.1
**Аудируемый документ:** предварительный план с 15 этапов + PWA

## Метод аудита

1. Каждый компонент → #0 Simplicity First: "есть ли стандартное простое решение?"
2. Industry benchmark: Superhuman, Shortwave, Sanebox, Gmail Smart Reply, Lindy, Mem.ai
3. Canon compliance — все 11 принципов + #0, жёстко
4. Reliability: где самая хрупкая точка? Что сломается первым?

## 8 находок

### #1 🔴 CRITICAL: LiteLLM в MVP — нарушение #0

**Что было:** LiteLLM docker + Redis + multi-model routing day 1

**Проблема:**
- +500 MB RAM, +docker-compose, +4 часа setup
- Anthropic SDK нативно уже даёт: prompt caching, streaming, batch API, rate limits

**Анти-паттерн #0:** "2 часа на permissions когда git push = 5 минут"

**Fix:** native Anthropic SDK + budget guard 30 строк. LiteLLM → Phase 2 (когда нужен 2-й provider).

---

### #2 🔴 CRITICAL: RAG в MVP — преждевременная оптимизация

**Что было:** LightRAG / sqlite-vss / Chroma в Этапе 3

**Проблема:**
- Для daily drafts не нужен (5 последних + profile = 95% качество)
- +embedding model 300 MB + vector index + сложный код

**Индустрия:** Shortwave / Superhuman добавили RAG в год 2.

**Fix:** SQLite FTS5 для MVP (встроено, 0 зависимостей). Embeddings → Phase 2 когда FTS5 упирается.

---

### #3 🟡 WARN: Full history ingest 3 года

**Что было:** full 3-year MTProto dump в Этапе 2

**Проблема:**
- ~150K сообщений × ~1500 API calls с паузами = 6-12ч фонового процесса
- FLOOD_WAIT / ban risk
- Ценность 0 для первой недели

**Индустрия:** все делают incremental + on-demand backfill.

**Fix:** только NEW messages с момента запуска. Backfill по запросу в Phase 2.

---

### #4 🟡 WARN: Два мозга (Qwen + Sonnet) сразу

**Что было:** cascade routing day 1

**Проблема:**
- 2 API ключа, router логика в каждом skill
- Debug в 2× сложнее

**Экономика:**
- Sonnet-only с caching: ~$20/мес
- Cascade: ~$14/мес
- Разница $6/мес vs сильная сложность

**Индустрия:** Superhuman стартовал с GPT-4 для всего. Cascade через 6 мес.

**Fix:** Sonnet 4.6 + prompt caching для всего. Qwen → Phase 2 когда видим распределение use.

---

### #5 🟡 WARN: Learning loop в MVP — нарушение #8

**Что было:** cron weekly auto-update voice_profile

**Проблема #8 Validate Before Automate:** сначала manually, потом автомат.
- В MVP мы ещё не знаем паттерны
- Автомат закрепит плохие раньше чем заметишь

**Fix:**
- MVP (2 недели): лог правок, 0 автомата
- Phase 2: автоматизация после 3+ стабильных паттерна замечено manually

---

### #6 🟡 WARN: Voice commands too ambitious — нарушение #5

**Что было:** 15 intent типов в MVP

**Проблема #5 Minimal Clear Commands:** "коридор без фантазий"
- 15 intent = модель путается, fail rate >30%

**Fix:** 3 команды максимум (Ответь/Правило/Поиск). Остальное — через inline-кнопки.

---

### #7 🟡 WARN: PWA в первоначальном плане

**Что было:** Фаза 2B через 2 недели

**Проблема:** PWA — UX улучшение, не функция. В MVP не добавляет новой capability (TG inline-кнопки дают то же).

**Индустрия:** Superhuman → iOS/Android через 6 мес. Shortwave аналогично.

**Fix:** MVP только TG (7-10 дней). PWA когда MVP доказан (trigger: "TG экран мал").

---

### #8 🟡 WARN: 15 этапов = too fine-grained

**Проблема:** false sense of progress. Индустриальные MVP = 3-5 milestones.

**Fix:** 6 этапов с ясной user value.

## Industry Patterns Reference

| Продукт | Стартовый стек | Когда добавили complexity |
|---------|---------------|---------------------------|
| Superhuman AI | GPT-4 + inline UI | Cascade через 6 мес |
| Shortwave | One bot, one model, summaries | RAG через 18 мес, drafts через 9 мес |
| Sanebox | Rule-based (не LLM) | LLM когда rules > 70% |
| Gmail Smart Reply | 3 фикс suggestions | Learning через 2 года |
| Lindy.ai | 1 skill | Multi-skill через 12 мес |
| Mem.ai | Plain text search | Embeddings через 1 год |

**Паттерн:**
1. Один канал сначала
2. Одна модель
3. Rules before ML
4. Drafts before auto-reply
5. Text before voice
6. RAG only when history > multiple GB
7. Automation only after 2-4 weeks manual

## Reliability Review — хрупкость компонентов

| Компонент | До audit | После audit |
|-----------|----------|-------------|
| gramjs session (ban) | средне | средне (не изменилось) |
| **LiteLLM proxy crash** | высоко ⚠️ | **0** (убрали) |
| Redis | средне | **0** (убрали) |
| Vector store downtime | средне | **0** (FTS5 in-SQLite) |
| Docker конфликты | средне | **0** (systemd only) |
| Cloud API errors | низко | низко + retry |
| **Ingest 3 лет timeouts** | высоко 🔴 | **0** (убрали) |
| **15 voice intents mis-parse** | высоко | низко (3 intent) |
| **Learning closed-loop bad patterns** | высоко | **0** (manual) |

**Сумма рисков упала на ~70%** при почти той же функциональности.

## Что сохранили из первоначального плана

- ✅ MTProto listener (gramjs)
- ✅ SQLite для памяти
- ✅ Отдельный bot для inline keyboards
- ✅ Grok STT переиспользование
- ✅ Изоляция от realty_*
- ✅ Heartbeat + Healthchecks.io dead-man switch
- ✅ Schedule 5/30 day/night
- ✅ Voice commands (но 3 вместо 15)
- ✅ Канон Алексея как руководство

## Что отложили из первоначального плана

- ⏭️ LiteLLM → Phase 2 (trigger: $≥$20/мес)
- ⏭️ Qwen/cascade → Phase 2 (trigger: Sonnet-only $≥$20/мес)
- ⏭️ RAG/embeddings → Phase 2 (trigger: FTS5 misses ≥50%)
- ⏭️ Full history 3-year ingest → on-demand
- ⏭️ Learning loop automation → после 2 недель manual
- ⏭️ Silero TTS → Phase 2 (trigger: 5+ использований кнопки)
- ⏭️ PWA → Phase 2 (trigger: "TG экран мал")
- ⏭️ Parser платных групп → Phase 3 (trigger: MVP стабилен 2 недели)
