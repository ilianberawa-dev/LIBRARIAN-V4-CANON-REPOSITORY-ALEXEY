**Статус:** RESERVE PLAN — не для немедленного применения
**Принцип:** делаем своё (план v1.2 не меняется). Если упрёмся — есть план Б через NotebookLM.

**Решение о fallback принимает ВЛАДЕЛЕЦ**, не архитектор.

---

## 📌 Что такое NotebookLM (краткая справка)

**Адрес:** notebooklm.google.com
**Цена:** Free (50 источников × 100 блокнотов) или Plus $20/мес (300 × 500)
**Тип:** Web-only SaaS, **нет API** (нельзя автоматизировать)
**Сильные стороны:** Q&A по PDF/YouTube/Web с цитированием, Audio Overview подкасты, Mind Maps, готовые форматы (Briefing/FAQ/Study Guide)
**Слабости:** не парсит Telegram, нет real-time, нет кастомизации, не интегрируется в код

## 🆘 Trigger conditions — когда рассмотреть NotebookLM

### Trigger 1: "Слепые зоны за 3 года" → реальное использование

**Если** через 6+ месяцев работы AI Assistant ты захочешь активный self-analysis по архиву, а наш FTS5 search дает <50% useful answers, **тогда:**

**План B:**
1. Voice команда "Export Maria last 6m" → bot генерит SQLite query → выгружает чат как Markdown файл
2. Bot отдаёт ссылку на скачивание (Telegram file)
3. Ты вручную загружаешь в свой NotebookLM notebook
4. Q&A через NotebookLM (citations, Audio Overview, Mind Map)

**Время реализации fallback:** 2 часа Stage 1.5 (новый export voice command)

**Что НЕ нужно строить если используем fallback:** embeddings, semantic search, complex self-analysis skill (-3 дня dev)

---

### Trigger 2: Изучение нового домена

**Если** тебе нужно изучить новую тему (realty Бали, индонезийское право, контент-маркетинг, n8n) — **не надо** просить AI Assistant парсить это.

**План B:**
1. Открой notebooklm.google.com
2. Создай новый блокнот "Realty Bali Research"
3. Загрузи 20-50 PDF / YouTube ссылок / web pages по теме
4. Audio Overview за 2 минуты → послушай в дороге
5. Q&A когда нужны конкретные факты

**Не надо строить:** парсер темы, RAG, knowledge graph, self-analysis по домену

---

### Trigger 3: Подготовка к встречам/контрактам

**Если** нужно прочитать 50-страничный договор или подготовиться к встрече с инвестором по 30 материалам:

**План B:**
1. Загрузи материалы в новый блокнот NotebookLM
2. Briefing doc → 1 страница executive summary
3. Q&A "найди риски / красные флаги / неоднозначные пункты"

**Время:** 10 минут vs неделя ручной обработки

---

### Trigger 4: Корпоративная knowledge base для команды

**Если** дойдёшь до Phase 4 SaaS с командой, и нужно "база знаний для onboarding":

**План B:**
1. Загрузи корпоративную документацию в NotebookLM (Plus tier $20/мес)
2. Share notebook с командой
3. Они задают вопросы вместо того чтобы спрашивать тебя

**Не надо строить:** корпоративный wiki, поисковую систему, FAQ bot

---

### Trigger 5: Изучение канала / эксперта

**Если** хочешь "выжать всю суть" из YouTube канала эксперта (Алексей Колесов, Lex Fridman, etc):

**План B:**
1. Скопируй URL'ы 20-30 видео канала в NotebookLM
2. Audio Overview = подкаст-дайджест за 30 минут
3. Mind Map покажет структуру тем
4. Briefing = одностраничный конспект

**Не надо строить:** парсер YouTube, транскрипция (NotebookLM делает сам), summarization pipeline

## 🔌 Возможные точки интеграции (вручную)

NotebookLM не имеет API, но можно облегчить ручной workflow:

### Идея 1: Export voice command в AI Assistant

Voice команда `"Export <chat_name> last <period> markdown"` → bot:
1. Запрашивает SQLite
2. Генерит .md файл с историей
3. Отправляет тебе как TG attachment
4. Ты вручную загружаешь в NotebookLM

**Время реализации:** 2 часа в Stage 5 (можно добавить как 5-ю voice команду опционально)

### Идея 2: Daily briefing exporter

Cron job вечером экспортирует summary дня → тебе в TG:
- Сколько сообщений
- Топ-3 темы дня
- Список ссылок которые упомянули в чатах

Если темы интересны → загрузить в NotebookLM для глубокого изучения.

### Идея 3: NotebookLM как "архив принятых решений"

Каждое важное решение, которое AI Assistant помог принять — экспортируешь в NotebookLM "Decisions Archive". Через год Audio Overview покажет паттерны твоих решений.

## ❌ Что НЕЛЬЗЯ делегировать в NotebookLM

| Функция нашего стека | Почему нельзя в NotebookLM |
|---------------------|---------------------------|
| Live TG event listener | Нет real-time + нет TG интеграции |
| Voice commands | Нет voice interaction в NotebookLM |
| Отправка сообщений собеседникам | NotebookLM только читает |
| Парсер каналов автоматический | NotebookLM требует мануальной загрузки |
| Парсер платных групп | NotebookLM не парсит TG в принципе |
| Контент-генерация (тексты, видео) | NotebookLM это reading assistant, не генератор |
| Personal data privacy | Загружая в Google = соглашаешься с их terms |
| Кастомные skills / workflows | NotebookLM закрытый продукт |
| Интеграция с n8n / Anthropic / другими API | Нет API у NotebookLM |
| Heartbeat / monitoring | Не имеет к NotebookLM отношения |

## 📋 Решение о применении fallback

Принимает **ВЛАДЕЛЕЦ** в одном из двух случаев:

1. **Активная разработка функции уперлась** — например self-analysis skill оказывается сложнее чем ожидалось → стоп, делегируем в NotebookLM, экономим неделю
2. **После MVP launch владелец видит, что некоторая фича недоиспользуется** → выпиливаем из нашего стека, переносим в NotebookLM workflow

**Решение архитектора без согласия владельца — НЕДОПУСТИМО.** Это документ-страховка, не план изменений.
